Frutimania – Resumen Ejecutivo

1.-Descripción del proyecto

Frutimania es una aplicación diseñada para la gestión de inventarios y control de precios en una frutería. El sistema permite registrar entradas y salidas de productos, consultar precios de mercado y apoyar en la toma de decisiones para vender a un precio competitivo. El proyecto incluye: -Aplicación Java -Pruebas JUnit -Administración del proyecto mediante GitHub Projects o Trello -Control de versiones con GitHub

Problema identificado
Las fruterías pequeñas carecen de sistemas organizados para: -Registrar inventario diario -Controlar pérdidas y caducidad -Consultar precios de mercado -Tomar decisiones informadas sobre ventas

Esto genera descontrol, pérdidas económicas y falta de estrategia competitiva.

Solución propuesta
Se propone desarrollar Frutimania, una aplicación que centraliza la gestión de productos mediante: -Registro digital de entradas y salidas -Actualización de stock en tiempo real -Integración con precios de mercado -Uso de metodologías ágiles con milestones, issues y tableros

Arquitectura de la solución
La arquitectura implementada sigue el siguiente flujo:

1.-Developer Realiza cambios en el código Java y los sube a GitHub.

2.-GitHub (Control de versiones) Administra branches, issues, milestones y cambios del repositorio.

3.-Aplicación Java Organizada con: /src/main/java /src/test/java
